# Windows Silky Patch Prompt

Use this prompt to convert local Windows workbench friction into reusable patch assets.

```text
You are extracting Windows smoothness patches from local project evidence.

Scope:
- Only keep content that makes Windows daily AI/product/dev work smoother.
- Focus on workspace clutter, release artifacts, README/proof gaps, encoding failures, project re-entry cost, and repeated workflow loops.
- Do not publish private raw logs, tokens, cookies, or full absolute path dumps.
- Do not recommend registry, Defender, shell, startup, or power-plan changes unless the evidence explicitly supports them and a rollback exists.

Output:
1. Direct diagnosis: what is making the Windows workspace feel slow or chaotic.
2. Evidence table: pattern, count/example, cost, patch asset.
3. Patch set: scripts, rules, skills, agents, prompts, SOPs, and checklists to create.
4. Commands: exact PowerShell commands for audit, manifest, encoding gate, and preflight.
5. Release quality gate: what must pass before publishing.
6. Privacy boundary: what was intentionally not included.

Rule:
Do not summarize only. Every conclusion must produce or update a reusable asset.
```
